# V0.2 Design — LangGraph Native Runtime + Checkpointer + Subagent

**Status:** approved 2026-05-29
**Supersedes:** Runtime Adapter (V0.1 hand-rolled state machine) and Subagent Runtime (V0.1 `architecture/future/subagent-runtime.md`)
**Target release:** v0.2.0

## TL;DR

V0.2 replaces the hand-rolled single-agent loop with a real LangGraph graph, adds a pluggable checkpointer (Sqlite default, Postgres adapter) for cross-process resume, and lands Subagent Runtime as a first-class capability. The work is a breaking refactor — V0.1 API contracts are not preserved.

## Goals

1. **Cross-process resume.** A run interrupted on host A can be resumed on host B by reading the shared checkpointer with the same `thread_id`.
2. **Production-grade persistence.** Run state survives process restarts. No reliance on in-memory dictionaries.
3. **Subagent Runtime.** Agents can delegate bounded sub-tasks to other agents via `delegate_to_<agent>` tools, with denied-action propagation in both directions and isolated workspaces.
4. **Streaming.** Harness API exposes `stream()` returning normalized `StreamEvent` iterator.

## Non-Goals

- HTTP server, web UI, hosted service.
- Multi-provider Model Adapter beyond `langchain-openai` (Anthropic / local stays V0.3+).
- Input Router (still deferred).
- Embedding-based memory.
- V0.1 API backward compatibility (project still in development; clean break authorized).

## Architecture Overview

```
Harness API
  └─ (per call) graph.invoke / .astream with thread_id
      └─ CompiledGraph (LangGraph)
          ├─ 8 nodes wrapping Modi steps
          ├─ BaseCheckpointSaver (Sqlite default | Postgres adapter | Memory for tests)
          ├─ interrupt() + Command(resume=) for approvals
          ├─ tools/subagent_tool dispatches a child CompiledGraph.invoke
          └─ trace middleware flushes state.pending_trace_events to trace.jsonl
```

### Invariants Preserved from V0.1

- Modi type contracts in `docs/types-reference.md`.
- Governance modules (`policy/`, `hooks/`, `tools/`, `output/`) are unchanged in their public API and called as pure functions inside graph nodes.
- Trace JSONL is independent storage; LangGraph checkpointer only stores state needed for resume.
- All untrusted-content wrapping, denied-retry, plan-mode rewriting remain semantically identical.

### Breaking Changes from V0.1

- `RuntimeAdapter._loop`, `_RunContext`, `_runs[run_id]` removed.
- `WorkspaceManager.save_state` / `snapshot_state` removed (checkpointer takes over). `index_workspace` kept.
- `ModiHarness.run_task` accepts `thread_id` and treats it as the LangGraph thread id; absence means a one-shot ephemeral thread.
- All introspection methods (`get_state` / `get_artifacts` / `get_trace` / `get_denials`) keyed by `thread_id`, not `run_id`.
- `ModiHarness.start_thread` removed.
- `approve_action` / `reject_action` now thin wrappers over `resume_task(Command(resume=...))`.
- `AgentState` adds three LangGraph-only fields (`pending_trace_events`, `repair_used`, `parent_thread_id`).
- `ToolSpec` adds `kind: "regular" | "subagent"` and `subagent_target: str | None`.
- `PermissionProfile` adds `allowed_subagents: list[str]` and `subagent_max_depth: int | None`.

## Detailed Design

### 1. Graph Structure

The main graph has 8 nodes:

```
START → load_agent → load_skills → build_context → model_step → route_response
                                                                      │
                                          ┌───────────────────────────┼─────────┐
                                          ↓                           ↓         ↓
                                    execute_tool              validate_output  FAIL
                                          │                           │
                                          ├──→ interrupt()            ├──→ build_context (repair)
                                          └──→ persist_state          ├──→ persist_state (validated/final)
                                                     │                └──→ FAIL (budget exhausted)
                                                     ↓
                                              should_continue?
                                                  ├──→ build_context
                                                  └──→ END
```

Node contracts:

- Signature: `(state: AgentState, config: RunnableConfig) -> dict` returning a partial state update.
- Side effects forbidden inside nodes (so LangGraph can replay); IO goes through the wrapper around node dispatch.
- Trace events are appended to `state["pending_trace_events"]`; a wrapper flushes them between nodes.
- Nodes may raise `langgraph.types.Interrupt(...)` for approval requests; LangGraph saves the state automatically.

### 2. AgentState Schema (with LangGraph reducers)

```python
from typing import Annotated
import operator

class AgentState(TypedDict):
    run_id: str
    root_run_id: str
    parent_run_id: str | None
    parent_thread_id: str | None
    thread_id: str
    agent_name: str
    permission_mode: PermissionMode
    task: dict[str, Any]
    messages: Annotated[list[Message], operator.add]
    loaded_skills: list[str]
    tool_calls: Annotated[list[ToolCallRecord], operator.add]
    denied_actions: Annotated[list[DeniedAction], operator.add]
    workspace_refs: Annotated[list[WorkspaceRef], operator.add]
    pending_approval: PendingApproval | None
    draft_output: dict[str, Any] | None
    final_output: dict[str, Any] | None
    step_count: int
    status: Literal["running", "interrupted", "blocked", "completed", "failed"]
    pending_trace_events: Annotated[list[TraceEvent], operator.add]
    repair_used: int
```

The default LangGraph serializer (`msgpack` + `pickle` fallback) handles all fields. Class instances (TraceRecorder, HookDispatcher) and IO handles must not be placed in state — they are re-instantiated inside the node from Settings carried via `RunnableConfig`.

### 3. Interrupt / Approval Flow

V0.1's bespoke `pending_approval / approve_action / reject_action` is replaced by LangGraph's `interrupt()` + `Command(resume=...)`:

```python
# inside execute_tool node, when policy returns require_approval:
user_decision = interrupt({
    "approval_id": new_ulid(),
    "tool_call_id": proposal["tool_call_id"],
    "summary": ...,
    "risk_level": ...,
})
# LangGraph saves the checkpoint here and pauses execution.
# On resume, user_decision is the payload passed to Command(resume=...).
if user_decision["decision"] == "approved":
    # execute the tool
else:
    # record DeniedAction
```

`approve_action(thread_id, approval_id, decision="approved")` is implemented as:

```python
graph.invoke(
    Command(resume={"approval_id": approval_id, "decision": "approved"}),
    config={"configurable": {"thread_id": thread_id}},
)
```

The thread_id alone is sufficient to resume; no in-memory context is required, satisfying the cross-process resume goal.

### 4. Checkpointer Abstraction

Modi uses `langgraph.checkpoint.base.BaseCheckpointSaver` directly. No Modi-defined abstraction layer.

`modi_harness.checkpoint.factory.build_checkpointer(settings)` returns one of:

- `MemorySaver` — tests only; production refuses.
- `SqliteSaver` — default, file at `MODI_CHECKPOINT_SQLITE_PATH` (`.modi/checkpoint.sqlite`).
- `PostgresSaver` — opt-in via `MODI_CHECKPOINT_BACKEND=postgres`; requires `MODI_CHECKPOINT_POSTGRES_DSN`.

New settings:

```text
MODI_CHECKPOINT_BACKEND=sqlite        # sqlite | postgres | memory
MODI_CHECKPOINT_SQLITE_PATH=.modi/checkpoint.sqlite
MODI_CHECKPOINT_POSTGRES_DSN=
MODI_SUBAGENT_MAX_DEPTH=3
```

Postgres adapter is a thin wrapper: validates DSN, lazy-imports `langgraph.checkpoint.postgres`, runs `setup()`. Pool size and other psycopg connection parameters can be expressed in the DSN itself (e.g. `?pool_min_size=...`); a dedicated Modi setting for pool size is deferred until we have a real Postgres deployment to size against.

### 5. Thread ID Naming Convention

| Source | thread_id format |
|---|---|
| `run_task(thread_id=None)` | `"run_<ULID>"` (ephemeral) |
| `run_task(thread_id="my_chat")` | `"my_chat"` (caller-supplied) |
| Subagent dispatch | `"<parent_thread_id>/sub/<ULID>"` |
| Nested subagent | `"<parent>/sub/<id>/sub/<id>"` |

Subagent depth is enforced by Policy Gate counting `/sub/` occurrences in `parent_thread_id`. Default cap is `MODI_SUBAGENT_MAX_DEPTH=3`.

### 6. Subagent Runtime

#### Auto-registered subagent tools

At ModiHarness construction, for each agent discovered by `AgentLoader`, register a `delegate_to_<agent_name>` tool:

```python
ToolSpec(
    name=f"delegate_to_{agent_name}",
    description=f"Delegate a bounded sub-task to the {agent_name} agent.",
    input_schema={
        "type": "object",
        "properties": {
            "task": {"type": "object"},
            "permission_mode": {"type": "string", "enum": ["ask", "auto", "plan", "bypass"]},
            "rationale": {"type": "string"},
        },
        "required": ["task", "rationale"],
    },
    risk_level="L2",          # dispatch risk; child tools enforce their own
    side_effect=True,
    kind="subagent",
    subagent_target=agent_name,
    ...
)
```

#### Tool Gateway dispatch branch

When `spec.kind == "subagent"`:

1. Schema-validate the child input.
2. Visibility: parent `allowed_subagents` filter.
3. Denied-retry guard against the child-call fingerprint.
4. `pre_tool_use` hooks (parent-level).
5. Policy Gate: L2 by default; additional checks for depth limit and permission_mode tightening.
6. Build child state:
   - `agent` = `spec.subagent_target`
   - `task` = arguments.task
   - `permission_mode` = arguments.permission_mode or parent.permission_mode (must be ≤ parent strictness)
   - `parent_thread_id`, `parent_run_id`, `root_run_id` propagated
   - `thread_id` = `f"{parent.thread_id}/sub/{new_ulid()}"`
   - `denied_actions` = copy of parent's denied_actions
7. Build child compiled graph (same `_graph_factory.build(agent_name)` used by main).
8. `child_graph.invoke(child_state, config={"configurable": {"thread_id": child_thread_id}})`.
9. Handle child status:
   - `completed` → tool_result = `{"output": child.final_output, "child_run_id": ...}`
   - `interrupted` → propagate interrupt to parent graph (parent also interrupts)
   - `failed` → tool_call recorded as error
   - `blocked` → tool_result = `{"draft": ..., "needs_review": True}`
10. Write back to parent state:
    - `parent.denied_actions += child.denied_actions diff`
    - `parent.workspace_refs += refs to child artifacts`
11. `post_tool_use` hooks.
12. Trace event includes `child_thread_id` and `trace_path`, not inlined child trace.

#### Trust boundary

Child output is **untrusted** to the parent. Tool Gateway wraps the child output as a `ContextBlock(trust="untrusted", source_kind="subagent_result")`, identical handling to regular tool results. Parent's OutputController runs the same untrusted-promotion and denied-side-effect checks.

#### Denied-action propagation

| Direction | Rule |
|---|---|
| Parent → Child (pre-dispatch) | Parent's `denied_actions` copied into child initial state. |
| Child → Parent (post-completion) | Diff of new `denied_actions` accrued in child appended to parent. |
| Sibling subagents | Not direct; sequential siblings inherit via the parent state. |

#### Permission mode constraints

Children may only be **as strict as or stricter than** parent, never more permissive. Strictness order: `bypass < auto < ask < plan`.

| Parent mode | Child default | Child may override to |
|---|---|---|
| `bypass` | `bypass` | `auto`, `ask`, or `plan` (any stricter) |
| `auto` | `auto` | `ask` or `plan` |
| `ask` | `ask` | `plan` only |
| `plan` | `plan` | (no stricter mode exists) |

A child request to relax (e.g. parent=`ask`, child requests `auto`) is denied at dispatch time by Policy Gate.

#### `allowed_subagents` opt-in

`AgentProfile.permission_profile.allowed_subagents`:

- absent / empty list → agent cannot dispatch any subagent (safe default)
- `["*"]` → any registered agent
- `["research-assistant", "case-reviewer"]` → only the listed agents

Context Manager filters `delegate_to_*` tools by this list before exposing them to the model.

#### Workspace layout

```
.modi/workspace/<root_run_id>/
├── input/
├── artifacts/
├── drafts/
├── logs/
│   ├── trace.jsonl
│   └── trace_<child_run_id>.jsonl   (parent points to but never inlines)
└── sub/
    └── <child_run_id>/
        ├── input/
        ├── artifacts/
        ├── drafts/
        └── logs/
            └── trace.jsonl
```

`WorkspaceManager` gains `create_child_run(parent_run_id, child_run_id)`. Path traversal checks treat the child's `sub/<id>/` as its own root.

### 7. Trace Collaboration

Nodes never call `TraceRecorder` directly. Each node appends events to `state["pending_trace_events"]`. The list field uses `operator.add` reducer, so it grows monotonically across the run — events are never dropped from state, ensuring deterministic replay.

A graph-level middleware flushes events to disk between transitions. Because the state list never shrinks, the middleware needs an out-of-state cursor to know which events have already been written:

- The middleware keeps a per-thread-id, per-process write cursor (in-memory).
- On each node transition, it iterates `state["pending_trace_events"]` from the cursor, calls `TraceRecorder.record()` per new event, advances the cursor.
- On resume in a fresh process, the cursor is rebuilt by reading the existing `trace.jsonl` and indexing by `event_id`. New events from `pending_trace_events` whose `event_id` is already in the file are skipped (idempotent under crash/resume).

trace.jsonl is append-only; concurrent writers from the same host are serialized by fcntl file lock (already in `WorkspaceManager.append_log`). State growth from accumulated `pending_trace_events` is bounded by the lifetime of one thread; for very long-lived threads, V0.3 may add a periodic compaction step.

### 8. ModiHarness API Surface

```python
class ModiHarness:
    # --- run lifecycle ---
    def run_task(*, agent, input, thread_id=None, permission_mode=None, options=None) -> RunTaskResponse: ...
    def resume_task(*, thread_id, payload=None) -> RunTaskResponse: ...
    def approve_action(*, thread_id, approval_id, decision="approved") -> RunTaskResponse: ...
    def reject_action(*, thread_id, approval_id, reason) -> RunTaskResponse: ...

    # --- introspection (keyed by thread_id) ---
    def get_state(*, thread_id) -> AgentState: ...
    def get_artifacts(*, thread_id) -> list[WorkspaceRef]: ...
    def get_trace(*, thread_id) -> Iterable[TraceEvent]: ...
    def get_denials(*, thread_id) -> list[DeniedAction]: ...
    def list_threads() -> list[ThreadInfo]: ...

    # --- memory (unchanged) ---
    def add_memory(record) -> MemoryRecord: ...
    def list_memory(*, scopes=None, types=None, tags=None) -> list[MemoryRecord]: ...
    def forget_memory(record_id) -> None: ...

    # --- thread ---
    def end_thread(thread_id) -> None: ...

    # --- hooks ---
    def list_hooks() -> list[HookSpec]: ...
    def get_hook_results(*, thread_id, event_id=None) -> list[HookResult]: ...

    # --- tools ---
    def register_tool(spec, handler, *, dry_run=None) -> None: ...

    # --- streaming ---
    def stream(*, agent, input, thread_id=None, permission_mode=None, options=None) -> Iterator[StreamEvent]: ...
```

Removed from V0.1: `start_thread`. `list_threads` delegates to checkpointer.

### 9. Streaming

`ModiHarness.stream()` wraps `graph.astream(... stream_mode="messages")` (or `astream_events`, whichever provides cleaner per-token deltas).

`StreamEvent.event_type` values populated in V0.2:

| event_type | source |
|---|---|
| `model_delta` | LangChain chat-model token stream |
| `tool_call_proposal` | model_step node output |
| `tool_call_started` | execute_tool node entry |
| `tool_call_result` | execute_tool node exit |
| `approval_request` | interrupt() callsite |
| `policy_decision` | projected from trace event of same type |
| `hook_dispatch` | projected from trace |
| `output_validation` | projected from trace |
| `terminal` | terminal node, payload includes RunTaskResponse |

### 10. Subagent Display in Context

Context Manager filters tools exposed to the model:

```
visible = agent.default_tools
        ∩ (agent.allowed_subagents projection of delegate_to_* tools)
        ∩ skill_union (existing rule)
        ∩ policy.visible_tools(agent, mode, state)
```

A parent agent without `allowed_subagents` sees zero `delegate_to_*` tools regardless of whether children are loaded.

## Testing Strategy

Existing 210 tests are rewritten under V0.2 semantics. New test categories:

### Cross-process resume e2e

`tests/runtime/test_cross_process_resume.py` uses `subprocess.run` to spawn a second Python process pointed at the same sqlite checkpointer:

1. Process A: run_task until interrupt. Persists state to sqlite.
2. Process A exits.
3. Process B: ModiHarness(checkpointer=sqlite). `approve_action(thread_id, approval_id)`.
4. Assert: response status `completed`; trace.jsonl shows continuous event sequence across processes.

### Subagent matrix

| Test | Asserts |
|---|---|
| Parent → child happy path | Child completes; parent receives output as untrusted block |
| Parent denied-action flows to child | Child propose-same fingerprint immediately denied |
| Child denied-action flows back to parent | Parent cannot propose the same action after child returns |
| Child interrupt escalates to parent | Parent graph.invoke returns interrupted with child thread_id |
| Approve child interrupt resumes both | Resume via child thread_id; on completion control returns to parent |
| Subagent depth limit | 4th level denied |
| `allowed_subagents=[]` denies dispatch | tool_call deny surfaces to model |
| Permission mode strict-only | Child attempting laxer mode denied |
| Untrusted child output | OutputController catches denied-side-effect claim of child output |

### Streaming smoke

| Test | Asserts |
|---|---|
| Stream tool-call run | Sequence: model_delta* → tool_call_proposal → tool_call_started → tool_call_result → model_delta* → terminal |
| Stream non-streaming equivalence | Terminal event payload equals `run_task()` response |

## Risks & Mitigations

| Risk | Mitigation |
|---|---|
| LangGraph API churn between minor versions | Pinned range in pyproject.toml; CI sanity on the pinned set; absorb in graph builder if needed. |
| Sqlite WAL multi-process write contention | Use WAL mode; document single-host assumption for sqlite backend; Postgres for true multi-host. |
| Trace duplication on resume | Middleware diffs by `event_id` against existing trace.jsonl before writing. |
| Subagent depth runaway | Hard cap `MODI_SUBAGENT_MAX_DEPTH`; Policy Gate denies past cap. |
| State bloat (denied_actions / messages append-only) | V0.2 keeps full history; V0.3 may add summarization. Acceptable for V0.2. |
| Pickle in checkpointer | Modi state is TypedDict of primitives; pickle is fallback only. No unsafe deserialization across untrusted sources. |

## Open Questions (deferred to implementation)

- Concrete `RunnableConfig` plumbing for passing Settings into nodes — likely `config["configurable"]["modi_settings"]`.
- Exact wire format for `Command(resume=...)` payload schema (likely a small TypedDict).
- Whether streaming should also expose a `subagent_started` / `subagent_completed` event_type alongside `tool_call_*`.

These are decided during N1–N3 implementation rather than blocking the spec.

2. `uv run pytest -m smoke` passes S1–S8 (including cross-process and subagent).
3. `modi run --agent X --task Y` works against the LangGraph runtime end-to-end.
4. A run interrupted in one Python process can be resumed by `approve_action` in a fresh Python process using the same sqlite file.
5. A sample agent with `allowed_subagents` configured can delegate to a child, child denials propagate back, depth limit enforced.
