# V0.4a Design Spec — Per-Agent Provider, Fallback, Error Normalization

**Status:** shipped
**Date:** 2026-05-29
**Shipped:** 2026-05-29
**Depends on:** V0.3 (shipped 2026-05-29)

---

## 1. Overview

V0.4a enhances the model layer with three features:

| Feature | Summary |
|---------|---------|
| Per-agent provider override | Each agent YAML can specify its own provider/model/api_key/base_url |
| Fallback | On transient failure after retries, try a configured secondary provider |
| Error normalization | Flat enum of error codes; provider exceptions mapped to normalized `ModelError` |

---

## 2. Per-Agent Provider Override

### 2.1 Agent YAML

Optional `model` block in frontmatter:

```yaml
model:
  provider: anthropic
  name: claude-sonnet-4-20250514
  api_key: ${ANTHROPIC_API_KEY}
  base_url: ""
  fallback:
    provider: openai
    name: gpt-4o
    api_key: ${OPENAI_API_KEY}
```

All fields optional. Missing fields inherit from global `ModelSettings`. The `${VAR}` syntax expands environment variables at parse time.

### 2.2 AgentProfile Extension

`AgentProfile["metadata"]["model"]` stores the parsed model config dict (after env expansion). Fields: `provider`, `name`, `api_key`, `base_url`, `fallback` (sub-dict with same fields minus `fallback`).

### 2.3 AgentLoader Changes

`_build_profile()` parses the `model` block from frontmatter. Env var expansion: any string value matching `${VAR_NAME}` is replaced with `os.environ.get("VAR_NAME", "")`. Store in `metadata["model"]`.

### 2.4 Per-Agent ModelAdapter Construction

New `models/cache.py` module:

```python
class ModelAdapterCache:
    def get_or_create(self, agent_model_config: dict | None, global_settings: ModelSettings) -> ModelAdapter
```

Caches adapters by `(provider, name, base_url)` tuple. If `agent_model_config` is None, returns the global adapter. Otherwise merges agent config over global defaults and constructs via `create_chat_model`.

### 2.5 Graph Node Integration

`model_turn_node` reads `profile["metadata"].get("model")` and calls `ModelAdapterCache.get_or_create(...)` to get the effective adapter. The cache lives on `GraphDeps`.

## 3. Fallback

### 3.1 Settings

`ModelSettings` gains:
- `fallback_provider: str = ""` — empty means no fallback
- `fallback_name: str = ""`
- `fallback_api_key: str = ""`
- `fallback_base_url: str = ""`

Env mappings: `MODI_MODEL_FALLBACK_PROVIDER`, `MODI_MODEL_FALLBACK_NAME`, `MODI_MODEL_FALLBACK_API_KEY`, `MODI_MODEL_FALLBACK_BASE_URL`.

### 3.2 Agent-Level Fallback

The `model.fallback` sub-block in agent YAML provides per-agent fallback config. Same fields as the primary model block (minus nested fallback — single hop only).

### 3.3 Behavior

After retries are exhausted on a transient error:
1. Check if fallback is configured (agent-level first, then global)
2. If yes, construct fallback model via `create_chat_model`
3. Attempt the same call once with the fallback model
4. Emit trace event: `model_fallback` with payload `{original_error, fallback_provider, fallback_name}`
5. If fallback succeeds, return its result (with `ModelResult.metadata["used_fallback"] = True`)
6. If fallback fails, raise the fallback's error as a `ModelError`

Fallback is a single hop. No chaining. No retry on the fallback attempt.

### 3.4 Transient Error Definition

Only these `ModelErrorCode` values trigger fallback:
- `TIMEOUT`
- `RATE_LIMITED`
- `SERVER_ERROR`

Non-transient errors (`AUTH_FAILED`, `CONTENT_FILTERED`, `CONTEXT_LENGTH_EXCEEDED`) never trigger fallback.

## 4. Error Code Normalization

### 4.1 ModelErrorCode Enum

```python
class ModelErrorCode(str, Enum):
    TIMEOUT = "timeout"
    RATE_LIMITED = "rate_limited"
    AUTH_FAILED = "auth_failed"
    CONTENT_FILTERED = "content_filtered"
    CONTEXT_LENGTH_EXCEEDED = "context_length_exceeded"
    SERVER_ERROR = "server_error"
    UNKNOWN = "unknown"
```

### 4.2 ModelError Exception

```python
class ModelError(Exception):
    code: ModelErrorCode
    original: Exception | None
    provider: str
    message: str
```

Raised by ModelAdapter when a model call fails (after retries and fallback are exhausted).

### 4.3 classify_error Function

```python
def classify_error(exc: Exception) -> ModelErrorCode
```

Maps provider exceptions to normalized codes:
- `TimeoutError`, `httpx.TimeoutException` → `TIMEOUT`
- HTTP 429, "rate limit" in message → `RATE_LIMITED`
- HTTP 401/403, "auth", "permission", "api key" → `AUTH_FAILED`
- "content filter", "safety", "blocked" → `CONTENT_FILTERED`
- "context length", "token limit", "max.*token" → `CONTEXT_LENGTH_EXCEEDED`
- HTTP 5xx, `ConnectionError` → `SERVER_ERROR`
- Everything else → `UNKNOWN`

Pattern matching is case-insensitive on exception message strings.

### 4.4 Integration with Retry/Fallback

The existing `_is_transient()` method in ModelAdapter is replaced by:
```python
_TRANSIENT_CODES = {ModelErrorCode.TIMEOUT, ModelErrorCode.RATE_LIMITED, ModelErrorCode.SERVER_ERROR}

def _is_transient(self, exc: Exception) -> bool:
    return classify_error(exc) in _TRANSIENT_CODES
```

This unifies the retry decision, fallback decision, and error reporting under one classification.

### 4.5 ModelResult Extension

`ModelResult` gains optional field:
- `fallback_used: bool = False` — True if the result came from the fallback provider

## 6. Non-Goals

- Multi-hop fallback chains
- Per-tool-call model selection
- Cost tracking or budget enforcement
- Model routing based on task complexity
- Streaming fallback (if primary stream fails mid-token, no recovery — just fail)
