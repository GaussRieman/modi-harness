# V0.3 Design Spec — Async Streaming, Multi-Provider, Memory Levels, Subagent Scenario

**Status:** draft  
**Date:** 2026-05-29  
**Depends on:** V0.2 (shipped 2026-05-29)

---

## 1. Overview

V0.3 delivers four independent features on top of the V0.2 LangGraph runtime:

| Milestone | Feature | Summary |
|-----------|---------|---------|
| N0 | Multi-Provider Model Adapter | Support OpenAI and Anthropic as configurable providers via a factory |
| N1 | Async Streaming | Per-token `model_delta` via `astream` on ModelAdapter, RuntimeAdapter, and ModiHarness |
| N2 | Memory Selection Levels | Configurable injection granularity: minimal / moderate / full |
| N3 | Subagent Sample Scenario | release-coordinator dispatches research-assistant end-to-end |

Dependency chain: N0 → N1 (async needs `acall`/`astream` on the adapter). N2 and N3 are independent of each other and of N0/N1.

---

## 2. N0 — Multi-Provider Model Adapter

### 2.1 Goal

Allow agents to use either OpenAI or Anthropic models by changing a single config field. No fallback chain — just provider selection.

### 2.2 New Module: `models/factory.py`

```python
def create_chat_model(settings: ModelSettings) -> BaseChatModel
```

Maps `settings.provider` to the correct LangChain class:

- `"openai"` → `ChatOpenAI` (from `langchain-openai`, already a dependency)
- `"anthropic"` → `ChatAnthropic` (from `langchain-anthropic`, new dependency)

Imports are lazy: `langchain-anthropic` is only imported when `provider == "anthropic"`. If the package is missing, raise `ModelConfigError` with an actionable message.

### 2.3 Settings Changes

`ModelSettings` gains:

- `provider: Literal["openai", "anthropic"]` — default `"openai"` (backwards-compatible)
- Env mapping: `MODI_MODEL_PROVIDER`

Existing fields (`name`, `api_key`, `base_url`) are passed through to whichever provider is selected. Both providers accept these via LangChain's constructor.

### 2.4 ModelAdapter Changes

New methods:

- `acall(pack: ContextPack, options: ModelCallOptions) -> ModelResponse` — async version of `call()`, uses `self._model.ainvoke(messages)`
- `astream(pack: ContextPack, options: ModelCallOptions) -> AsyncIterator[str]` — yields token strings from `self._model.astream(messages)`, extracting `chunk.content` from each `AIMessageChunk`

The sync `call()` method is unchanged.

### 2.5 ModiHarness.__init__ Change

Replace direct `ChatOpenAI(...)` construction with `ModelFactory.create(settings.model)`.

### 2.6 New Dependency

`langchain-anthropic>=0.3` added to `pyproject.toml` main dependencies.

### 2.7 Errors

New `ModelConfigError` in `models/errors.py`:
- Raised when provider package is not installed
- Raised when required fields (api_key) are missing for the selected provider

## 3. N1 — Async Streaming

### 3.1 Goal

Add an async streaming path that yields per-token `model_delta` events. The existing sync `stream()` (whole-turn granularity) remains unchanged.

### 3.2 ModelAdapter.astream

```python
async def astream(self, pack: ContextPack, options: ModelCallOptions) -> AsyncIterator[str]:
    async for chunk in self._model.astream(messages):
        if chunk.content:
            yield chunk.content
```

### 3.3 Graph Node: model_turn_node Async Variant

The `model_turn_node` currently calls `model.call(pack, options)`. For async streaming, the graph must be invoked via `graph.astream()`. LangGraph's async streaming natively supports async node functions.

Add `async_model_turn_node` that calls `model.acall()` (not `astream` — per-token streaming is handled at the RuntimeAdapter level by intercepting LangGraph's stream events, not inside the node itself).

### 3.4 RuntimeAdapter.astream

```python
async def astream(self, request: RunTaskRequest) -> AsyncIterator[dict]:
```

Wraps `self._graph.astream(state, config=config, stream_mode="messages")`. The `"messages"` stream mode yields per-token `AIMessageChunk` events from the model node.

Projects events into the same `StreamEvent` shape:
- `model_delta` — `{"event_type": "model_delta", "delta": "<token>", "thread_id": "..."}`
- `tool_call_proposal` — emitted when a complete tool call is staged
- `tool_call_result` — emitted after tool execution
- `approval_request` — emitted on interrupt
- `terminal` — final event carrying the full `RunTaskResponse`

### 3.5 ModiHarness.astream

```python
async def astream(
    self,
    *,
    agent: str,
    input: str,
    thread_id: str | None = None,
    permission_mode: str | None = None,
    options: dict | None = None,
) -> AsyncIterator[dict]:
```

Public async API. Mirrors the sync `stream()` signature exactly.

### 3.6 Sync stream() Unchanged

The existing `RuntimeAdapter.stream()` and `ModiHarness.stream()` remain sync, whole-turn granularity. No changes.

## 4. N2 — Memory Selection Levels

### 4.1 Goal

Replace the hardcoded memory selection priority with a configurable granularity dial — `minimal`, `moderate`, or `full` — that controls how much memory is injected into context.

### 4.2 New Type

```python
MemoryLevel = Literal["minimal", "moderate", "full"]
```

Added to `types.py`.

### 4.3 Level Definitions

| Level | Types included | Token budget |
|-------|---------------|--------------|
| `minimal` | feedback | 500 |
| `moderate` | feedback, user, project | 1500 |
| `full` | feedback, user, project, reference | 3000 |

Within each level, priority ordering is preserved: `feedback > user > project > reference`.

### 4.4 Agent YAML

New optional frontmatter field:

```yaml
memory_level: moderate  # default if omitted
```

### 4.5 Per-Run Override

`RunTaskRequest.options` gains an optional `memory_level` field. If set, it overrides the agent's default for that run.

### 4.6 MemoryStore.select_for_context Changes

Current signature:

```python
def select_for_context(self, *, scope, task_tags, reference_keys, agent_name) -> list[MemoryRecord]
```

New signature adds `level: MemoryLevel = "moderate"`:

```python
def select_for_context(self, *, scope, task_tags, reference_keys, agent_name, level: MemoryLevel = "moderate") -> list[MemoryRecord]
```

Internally, the method filters types and applies the token budget based on the level table above.

### 4.7 Context Manager Integration

`ContextManager.build_context_pack()` reads the effective memory level (from run options or agent profile) and passes it to `select_for_context()`.

## 5. N3 — Subagent Sample Scenario

### 5.1 Goal

Prove the V0.2 subagent runtime works end-to-end with real agent definitions by having release-coordinator delegate to research-assistant.

### 5.2 Agent Changes

**release-coordinator** (`docs/agents/release-coordinator/agent.md`):
- Add to permission profile: `allowed_subagents: [research-assistant]`
- Add to tools: `delegate_to_research_assistant`

No changes to research-assistant — it's the child and doesn't need to know it's being delegated to.

### 5.3 New Scenario

`docs/scenarios/release-coordinator-with-research/`:

- `scenario.md` — describes the delegation flow
- `task.json`:
  ```json
  {
    "input": "Prepare release notes for v2.5. Research what breaking changes were introduced in our upstream dependency 'libcore' between v3.1 and v3.2, then summarize them in the release notes draft.",
    "agent": "release-coordinator",
    "permission_mode": "auto"
  }
  ```
- `expected.md` — output includes:
  - Release notes structure from release-coordinator
  - Research findings sourced from research-assistant (marked untrusted in trace)
  - Delegation visible in trace as `delegate_to_research_assistant` tool call

### 5.4 Smoke Test S9

New test in `tests/test_smoke_scenarios.py`:

- Scripted model for release-coordinator produces a `delegate_to_research_assistant` tool call
- Scripted model for research-assistant produces research findings
- Assertions:
  - Parent run completes with status `completed`
  - Trace contains a `tool_call` event with `tool_name == "delegate_to_research_assistant"`
  - Child thread_id follows `<parent>/sub/<ulid>` pattern
  - Child output is wrapped as untrusted in parent's context
  - Final output includes content from both agents

## 6. Testing Strategy

Each milestone follows TDD:

- **N0:** Unit tests for factory (both providers + missing package error), integration test for `acall`
- **N1:** Async streaming tests using `pytest-asyncio` — token-level delta events, terminal equivalence with sync path
- **N2:** Unit tests for each level's filtering behavior, integration test with context manager
- **N3:** Smoke test S9 (full parent→child→parent with scripted models)

Total new tests estimated: ~15–20.

## 8. Non-Goals

- Fallback/retry logic between providers
- Local model support (Ollama/vLLM) — deferred to V0.4
- Vector memory backend — explicitly excluded from V0.3
- Streaming propagation from child subagent to parent's stream — deferred
- Per-agent provider override (all runs use the global provider setting)
