# V0.4c Plugin System — Implementation Plan

**Spec:** [`docs/superpowers/specs/2026-05-29-v0.4c-plugin-system-design.md`](../specs/2026-05-29-v0.4c-plugin-system-design.md)

**Goal:** Add a plugin system so third parties can publish `pip install`-able packages contributing agents, skills, and tools to Modi Harness.

**Tech:** `importlib.metadata` (stdlib), existing stack. **Hard rule: never pin langgraph to an older line — always ride latest.**

**Test discipline:** TDD per task. Each task = write failing test → minimal impl → green → commit. Push at end of each milestone.

## N0 — Discovery Module

- [ ] **N0.1** Create `src/modi_harness/plugins.py` with `PluginInfo` TypedDict (fields: `name: str`, `agents_dir: Path | None`, `skills_dir: Path | None`, `tools: list[tuple[dict, Callable]]`, `source: str`) and `PluginLoadError(Exception)` with attributes `plugin_name: str`, `source: str`, `message: str`. Commit.

- [ ] **N0.2** Add `_validate_plugin_dict(d: dict, source: str) -> PluginInfo` helper to `plugins.py`. Validation rules per spec §3.4: `name` is non-empty string; `agents_dir` (if provided) exists as directory; `skills_dir` (if provided) exists as directory; `tools` (if provided) is a list of `(dict, callable)` tuples where each spec dict has `name`, `description`, `input_schema`, `risk_level`. Raises `PluginLoadError` with descriptive message. TDD via `tests/plugins/test_discovery.py::test_validate_*` (5+ cases: ok, missing name, bad agents_dir, bad tool spec, etc.).

- [ ] **N0.3** Add `discover_plugins() -> list[PluginInfo]` to `plugins.py`. Uses `importlib.metadata.entry_points(group="modi_harness.plugins")`. For each entry point: load the callable (catch ImportError/ModuleNotFoundError/AttributeError → raise PluginLoadError); call it (catch any exception → PluginLoadError); validate via `_validate_plugin_dict`; build PluginInfo with `source = f"entry_point:{ep.dist.name} v{ep.dist.version}"`. Returns the list in iteration order. TDD via `tests/plugins/test_discovery.py::test_discover_*` using `monkeypatch` on `importlib.metadata.entry_points` with mock entry points.

- [ ] **N0.4** Create `tests/plugins/fixtures/sample_plugin/` package with:
  - `__init__.py` exporting `get_plugin()` returning a valid dict
  - `agents/sample-agent.md` (minimal frontmatter)
  - `skills/sample-skill/SKILL.md` (minimal frontmatter)
  - One sample tool `(spec_dict, lambda **_: None)` in the returned dict
  Verify importable from tests; commit.

- [ ] **N0.5** Add fail-fast tests: scripted entry point that raises ImportError; one whose function raises at call time; one returning invalid dict. Each must raise `PluginLoadError` with the right `plugin_name` and source. Commit.

- [ ] **N0.6** Full suite green; push.

**Exit gate:** `discover_plugins()` returns `list[PluginInfo]` for valid plugins, raises `PluginLoadError` on any failure; sample fixture loads cleanly.

## N1 — Harness Integration

- [ ] **N1.1** Add parameters to `ModiHarness.__init__`: `plugins: list[PluginInfo] | None = None` and `discover_plugins: bool = True`. If `plugins` is None and `discover_plugins=True`, call `discover_plugins()` (the function, not the param — pick a different param name to avoid shadowing). Use `auto_discover_plugins: bool = True` as the param name. Commit.

- [ ] **N1.2** Wire plugin agent dirs into `AgentLoader`: collect all non-None `agents_dir` paths from plugins; pass to `AgentLoader(project_dir=agents_dir, plugin_dirs=collected)`. Same for skills. Existing `AgentLoader` and `SkillLoader` already accept `plugin_dirs`.

- [ ] **N1.3** Register plugin tools after the registry exists but before `_register_subagent_tools()`. Iterate `plugins`, then `plugin["tools"]`, call `self.register_tool(spec, handler)`. Tool name collisions raise the existing `ToolDuplicateError`.

**Exit gate:** Harness auto-discovers plugins by default; explicit `plugins` parameter overrides discovery; `auto_discover_plugins=False` disables; failures fail-fast at construction.

## Risks watch

- **`importlib.metadata` API drift across Python versions** — `entry_points(group=...)` was added in 3.10; we require 3.11+ so safe. Test on 3.11 and 3.12.
- **Plugin tool name collisions** — fail-fast via existing `ToolDuplicateError`. Acceptable; users see the conflict immediately.
- **Subagent tool ordering** — plugins register tools BEFORE subagent tools, so plugin agents auto-get `delegate_to_<plugin-agent>` correctly.
- **Test isolation** — entry points are global state; tests must use `monkeypatch` to mock `entry_points()` calls, never rely on real installed plugins.
