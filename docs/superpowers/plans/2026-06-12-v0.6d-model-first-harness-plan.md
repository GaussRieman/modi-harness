# V0.6.d Model-First Harness — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move Harness usage guidance out of the research-assistant agent prompt and into builtin tool descriptions, so agent authors write domain behavior only (spec §4.4, §8).

**Architecture:** Builtin tool `description` strings become the single home for "how to use Memory/Workspace" guidance (recall-before-acting, proposals vs durable writes, memory-is-not-an-output-store). The example agent prompt is reframed to domain behavior + output expectations. No runtime/graph rewrite (spec §9). The example's offline memory test drives the model with scripted tool calls, so it is independent of prompt wording and must keep passing unchanged.

**Tech Stack:** Python 3.12, pytest, modi_harness (`src/modi_harness`), uv.

**Spec:** `docs/superpowers/specs/2026-06-12-v0.6d-model-first-harness-design.md`

**Explicitly out of scope (faithful to spec §9 "no large runtime rewrite"):** automatic memory preselection (§6 Memory uses "may"), context-assembly changes (§6 Context — lazy workspace already shipped). Record these as follow-ups in the changelog, do not implement here.

```python
from __future__ import annotations

from pathlib import Path

_AGENT_MD = (
    Path(__file__).resolve().parents[2]
    / "examples" / "research_assistant" / "agents" / "research-assistant.md"
)


def _body() -> str:
    text = _AGENT_MD.read_text(encoding="utf-8")
    # Strip the YAML frontmatter (between the first two '---' lines).
    parts = text.split("---", 2)
    assert len(parts) == 3, "expected YAML frontmatter delimited by ---"
    return parts[2]


def test_prompt_body_has_no_harness_internals():
    body = _body()
    for term in ("scope", "trace", "artifact", "draft", "propose_memory", "save_memory"):
        assert term not in body, f"agent prompt should not mention Harness internal {term!r}"


def test_prompt_body_keeps_domain_behavior():
    body = _body()
    assert "研究" in body          # domain: research
    assert "证据" in body          # domain: evidence
    assert "source-evaluation" in body  # references its skills
```
