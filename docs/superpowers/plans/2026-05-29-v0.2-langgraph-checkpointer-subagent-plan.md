# V0.2 LangGraph Native Runtime + Checkpointer + Subagent — Implementation Plan

**Spec:** [`docs/superpowers/specs/2026-05-29-v0.2-langgraph-checkpointer-subagent-design.md`](../specs/2026-05-29-v0.2-langgraph-checkpointer-subagent-design.md)

**Goal:** Replace the V0.1 hand-rolled loop with a real LangGraph graph backed by a pluggable checkpointer, land Subagent Runtime, and expose a streaming API.

**Tech:** Latest `langgraph` (1.2.x at time of writing), `langgraph-checkpoint-sqlite>=2`, `langgraph-checkpoint-postgres` (lazy), `langchain>=1.0`, existing stack. **Hard rule: never pin langgraph to an older line — always ride latest.**

**Test discipline:** TDD per task. Each task = write failing test → minimal impl → green → commit. Push at end of each milestone.

## N1 — LangGraph Main Graph

- [ ] **N1.1** Create `graph/state.py`: re-export `AgentState` as `MainGraphState` (or alias). Sanity test asserting it's the same TypedDict instance the rest of the codebase uses.
- [ ] **N1.2** Create `graph/nodes.py`: extract the 8 step functions out of the V0.1 `_loop` — `load_agent`, `load_skills`, `build_context`, `model_step`, `route_response`, `execute_tool`, `validate_output`, `persist_state`. Each takes `(state, config) -> dict` (partial state). No IO inside nodes; they append `TraceEvent`s to `state["pending_trace_events"]` and call governance modules as pure functions. Deps (loaders, gateway, policy, etc.) are pulled from `config["configurable"]["modi_deps"]`. Unit tests per node under `tests/graph/test_nodes.py`.
- [ ] **N1.3** Create `graph/trace_middleware.py`: a per-thread, per-process cursor that drains `state["pending_trace_events"]` to `TraceRecorder` between node transitions. On resume in a fresh process, rebuild cursor by reading existing `trace.jsonl` and skipping events whose `event_id` is already present. Tests: cursor advances; duplicate `event_id` suppressed; idempotent under crash/resume.
- [ ] **N1.4** Create `graph/builder.py`: `build_main_graph(deps, checkpointer) -> CompiledGraph`. Wires the 8 nodes per the spec diagram, attaches the checkpointer, registers the trace middleware as `StateGraph` post-node hook. Test: compiled graph exposes the expected node set; `get_state(config)` after a no-op invoke returns the seed state.
- [ ] **N1.5** Implement `interrupt()` in `execute_tool`: when Policy Gate returns `require_approval`, raise `langgraph.types.Interrupt(payload=PendingApproval-shaped dict)`. Unit test confirms the graph pauses and `graph.get_state` shows `pending_approval` set.
- [ ] **N1.6** Rewrite `runtime/adapter.py`: remove `_loop`, `_RunContext`, `_runs`, `approve`, `reject`. New `RuntimeAdapter` holds the compiled graph and exposes `run(input) -> RunTaskResponse`, `resume(thread_id, payload) -> RunTaskResponse`. `run` calls `graph.invoke` with `config={"configurable": {"thread_id": ..., "modi_deps": ...}}`; `resume` uses `Command(resume=payload)`. Rewrite `tests/runtime/test_loop.py` against the new shape (6 cases preserved semantically).
- [ ] **N1.7** Rewrite `api/harness.py`: drop `start_thread`; key all introspection (`get_state`, `get_artifacts`, `get_trace`, `get_denials`) by `thread_id`; add `resume_task(thread_id, payload)`; `approve_action` / `reject_action` become thin wrappers over `resume_task` building `Command(resume={"approval_id": ..., "decision": ...})`. `list_threads` delegates to `checkpointer.list()` / equivalent. Rewrite `tests/test_harness.py`.
- [ ] **N1.8** Strip `WorkspaceManager.save_state` and `snapshot_state`; keep `index_workspace`, `create_run`, `append_log`. Drop the snapshot tests in `tests/workspace/test_manager.py`.
- [ ] **N1.9** Update `__main__.py` CLI: `modi run` now treats `--thread-id` as primary; `modi resume --thread-id X` added.
- [ ] **N1.10** Full suite green; push.

**Exit gate:** Every previously passing V0.1 test now drives the LangGraph runtime via the rewritten facade. `_runs` dict is gone from the codebase (`grep` confirms). `interrupt()` + `Command(resume=)` round-trip works in-process.

## N2 — Subagent Runtime

- [ ] **N2.1** Implement `subagent/dispatcher.py`: `dispatch_subagent(spec, args, parent_state, deps) -> ToolDispatchResult`. Steps per spec §6: schema-validate child input, visibility check against parent `allowed_subagents`, denied-retry guard, parent `pre_tool_use` hooks, Policy Gate subagent decision (depth + permission_mode tightening), build child state with propagated `parent_thread_id`/`parent_run_id`/`root_run_id`/`denied_actions`, child `thread_id = f"{parent_thread_id}/sub/{ulid}"`, invoke child compiled graph, handle child status (`completed` / `interrupted` / `failed` / `blocked`), propagate child `denied_actions` diff and `workspace_refs` back to parent, run parent `post_tool_use` hooks, emit trace event with `child_thread_id` (no inline child trace). Unit tests in `tests/subagent/test_dispatcher.py`.
- [ ] **N2.2** Auto-register subagent tools at `ModiHarness.__init__`: for each agent discovered by `AgentLoader`, register `delegate_to_<name>` ToolSpec with `kind="subagent"`, `subagent_target=<name>`, `risk_level="L2"`, `side_effect=True`, input schema `{task, permission_mode?, rationale}`. Test: harness instantiated with N agent dirs has N matching `delegate_to_*` entries in the registry.
- [ ] **N2.3** Wire the subagent branch into `tools/gateway.py`: when `spec["kind"] == "subagent"`, route to `dispatch_subagent` instead of the registered handler; preserve the existing `ToolDispatchResult` contract so `execute_tool` node is agnostic. Unit test: gateway dispatches to subagent path for `kind="subagent"` entries.
- [ ] **N2.4** Extend `policy/gate.py` with two new rules: (a) depth check — deny if `parent_thread_id.count("/sub/") >= effective_max_depth`; (b) permission-mode tightening — deny if requested child mode is laxer than parent (strictness order `bypass < auto < ask < plan`). Effective max depth = `agent.permission_profile.subagent_max_depth or settings.subagent.max_depth`. Unit tests for both rules.
- [ ] **N2.5** Add `WorkspaceManager.create_child_run(parent_run_id, child_run_id)`: lays out `.modi/workspace/<root>/sub/<child>/{input,artifacts,drafts,logs}`. Path traversal checks treat `sub/<id>/` as its own root. Unit test.
- [ ] **N2.6** Context Manager filters `delegate_to_*` tools by `agent.permission_profile.allowed_subagents` before exposing to the model (`[]` → none, `["*"]` → all, list → intersect). Extend existing `tests/context/test_manager.py`.
- [ ] **N2.7** Subagent trust boundary: child output wrapped as `ContextBlock(trust="untrusted", source_kind="subagent_result")` before the parent's `OutputController` sees it. Test that parent's denied-side-effect / untrusted-promotion checks fire on synthetic child output.
- [ ] **N2.8** Implement the 9 e2e scenarios from spec §Testing-Strategy Subagent-Matrix in `tests/subagent/test_e2e.py`:
  1. Parent → child happy path.
  2. Parent denied-action flows to child (child propose-same fingerprint → immediate deny).
  3. Child denied-action flows back to parent (parent cannot re-propose after child returns).
  4. Child interrupt escalates to parent.
  5. Resume child interrupt → control returns to parent on child completion.
  6. Subagent depth limit (4th level denied).
  7. `allowed_subagents=[]` denies dispatch (surfaces as tool deny to model).
  8. Child requests laxer permission mode → denied.
  9. Untrusted child output → OutputController catches denied-side-effect claim.
- [ ] **N2.9** Full suite green; push.

**Exit gate:** All 9 subagent e2e tests pass with the canned scripted model. Denied-action propagation bi-directional. Depth + mode-tightening enforced by Policy Gate, not by ad-hoc checks in the dispatcher.

## N3 — Streaming

- [ ] **N3.1** Add `ModelAdapter.astream(pack) -> AsyncIterator[StreamEvent]`: wrap LangChain `chat_model.astream` emitting `model_delta` events for each token chunk, then a final `model_result`-ish event. Unit test with a recorded `BaseChatModel` that yields chunks.
- [ ] **N3.2** Add `ModiHarness.stream(*, agent, input, thread_id=None, permission_mode=None, options=None) -> Iterator[StreamEvent]`. Wraps `graph.astream(... stream_mode="messages")` (or `astream_events`, whichever yields cleaner per-token deltas). Project graph events into Modi `StreamEvent`s: `model_delta`, `tool_call_proposal`, `tool_call_started`, `tool_call_result`, `approval_request` (from interrupt), and synthesize `terminal` carrying the final `RunTaskResponse`. Project `policy_decision`, `hook_dispatch`, `output_validation` from in-state trace events of the same type.
- [ ] **N3.3** Streaming smoke tests in `tests/streaming/test_stream.py`:
  - **Stream tool-call run**: assert sequence `model_delta*` → `tool_call_proposal` → `tool_call_started` → `tool_call_result` → `model_delta*` → `terminal`.
  - **Equivalence**: terminal payload equals what `run_task(...)` would return for the same input.
- [ ] **N3.4** Full suite green; push.

**Exit gate:** `stream()` returns a normalized iterator; smoke run with the scripted model produces the canonical event order; terminal event matches non-streaming response.

## Risks watch (during execution)

- **LangGraph API surface drift inside 0.2.x** — if minor releases shift `interrupt`/`Command` semantics, pin tighter and absorb in `graph/builder.py`.
- **Sqlite multi-process contention** — checkpointer opens with `check_same_thread=False`; WAL mode enabled by SqliteSaver. If S7 flakes on macOS, switch to file-lock around `invoke`.
- **Trace duplication on resume** — middleware MUST dedupe by `event_id` against on-disk `trace.jsonl` before writing. N1.3 covers this; revisit if S7 shows duplicates.
- **State bloat from append-only reducers** — accepted for V0.2; V0.3 may add periodic compaction.
