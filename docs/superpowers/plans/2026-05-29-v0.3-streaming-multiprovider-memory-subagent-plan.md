## N2 — Memory Selection Levels

- [x] **N2.1** Add `MemoryLevel = Literal["minimal", "moderate", "full"]` to `types.py`. Commit.
- [x] **N2.2** Refactor `MemoryStore.select_for_context` to accept `level: MemoryLevel = "moderate"`. Level controls which types are included and the token budget: `minimal` = feedback only / 500 tokens; `moderate` = feedback+user+project / 1500 tokens; `full` = all types / 3000 tokens. TDD via `tests/memory/test_levels.py` (3 cases: one per level verifying type filtering and budget).
- [x] **N2.3** Parse `memory_level` from agent YAML frontmatter in `AgentLoader`; default `"moderate"` if absent. Store in `AgentProfile.metadata["memory_level"]`. Extend `tests/agents/test_loader.py`.
- [x] **N2.4** `ContextManager.build_context` reads effective memory level from `state.get("options", {}).get("memory_level")` or `agent.metadata.get("memory_level", "moderate")` and passes it to `select_for_context`. Extend `tests/context/test_manager.py`.
- [x] **N2.5** `model_turn_node` propagates `memory_level` through to context build (already flows via state → context manager). Verify with existing graph tests.
- [x] **N2.6** Full suite green; push.

**Exit gate:** Memory injection respects the level dial; agents can declare a default; callers can override per-run via `options.memory_level`.

## Risks watch

- **langchain-anthropic version drift** — if `ChatAnthropic` changes constructor kwargs, absorb in `factory.py`.
- **LangGraph `astream` mode differences** — `stream_mode="messages"` may behave differently across providers; test with both mocked OpenAI and Anthropic chunks.
- **Memory level budget edge cases** — a single large feedback record could exhaust the `minimal` budget; accepted for V0.3.
