# V0.4a Model Layer — Per-Agent Provider, Fallback, Error Normalization — Implementation Plan

**Spec:** [`docs/superpowers/specs/2026-05-29-v0.4a-model-layer-design.md`](../specs/2026-05-29-v0.4a-model-layer-design.md)

**Goal:** Let each agent pick its own model provider, fall back to a secondary on transient failure, and normalize all provider errors into a flat enum.

**Tech:** `langchain-openai>=1.0`, `langchain-anthropic>=0.3`, existing stack. **Hard rule: never pin langgraph to an older line — always ride latest.**

**Test discipline:** TDD per task. Each task = write failing test → minimal impl → green → commit. Push at end of each milestone.

## N0 — Error Code Normalization

- [ ] **N0.1** Add `ModelErrorCode(str, Enum)` to `models/errors.py` with 7 values: `TIMEOUT`, `RATE_LIMITED`, `AUTH_FAILED`, `CONTENT_FILTERED`, `CONTEXT_LENGTH_EXCEEDED`, `SERVER_ERROR`, `UNKNOWN`. Add `ModelError(Exception)` with fields `code`, `original`, `provider`, `message`. Commit.
- [ ] **N0.2** Add `classify_error(exc: Exception) -> ModelErrorCode` to `models/errors.py`. Pattern-matches on exception type and message string (case-insensitive): `TimeoutError`/`httpx.TimeoutException` → TIMEOUT; "429"/"rate limit" → RATE_LIMITED; 401/403/"auth"/"api key" → AUTH_FAILED; "content filter"/"safety"/"blocked" → CONTENT_FILTERED; "context length"/"token limit" → CONTEXT_LENGTH_EXCEEDED; 5xx/`ConnectionError` → SERVER_ERROR; else UNKNOWN. TDD via `tests/models/test_error_codes.py` (7+ cases, one per code).
- [ ] **N0.3** Replace `_is_transient()` in `models/adapter.py` with: `classify_error(exc) in {TIMEOUT, RATE_LIMITED, SERVER_ERROR}`. Existing retry tests still green.
- [ ] **N0.4** Update `models/__init__.py` to re-export `ModelErrorCode`, `ModelError`, `classify_error`.
- [ ] **N0.5** Full suite green; push.

**Exit gate:** All provider exceptions map to a normalized code; retry logic uses `classify_error`; existing 282+ tests green.

## N1 — Fallback

- [ ] **N1.1** Add fallback fields to `ModelSettings` in `config/settings.py`: `fallback_provider: str = ""`, `fallback_name: str = ""`, `fallback_api_key: str = ""`, `fallback_base_url: str = ""`. Add env mappings `MODI_MODEL_FALLBACK_PROVIDER`, `MODI_MODEL_FALLBACK_NAME`, `MODI_MODEL_FALLBACK_API_KEY`, `MODI_MODEL_FALLBACK_BASE_URL`. Commit.
- [ ] **N1.2** Add `fallback_used: bool` field to `ModelResult` in `types.py` (default `False`). Commit.
- [ ] **N1.3** Extend `ModelAdapter.__init__` to accept optional `fallback_config: dict | None = None` (keys: `provider`, `name`, `api_key`, `base_url`). Store as `self._fallback_config`.
- [ ] **N1.4** Implement fallback in `ModelAdapter._with_retry`: after retries exhausted on a transient error, if `self._fallback_config` is set, construct fallback model via `create_chat_model(...)`, call it once. If it succeeds, return result with `fallback_used=True`. If it fails, raise `ModelError` with the fallback's error. Same for `_with_retry_async`. TDD via `tests/models/test_fallback.py` (cases: fallback succeeds after primary fails, fallback also fails, no fallback configured → just raises, non-transient error → no fallback attempted).
- [ ] **N1.5** Emit trace event `model_fallback` when fallback is used: append to `pending_trace_events` in the graph node after `model.call()` returns with `fallback_used=True`. Payload: `{original_error, fallback_provider, fallback_name}`.
- [ ] **N1.6** Full suite green; push.

**Exit gate:** Fallback fires on transient failure; single hop only; trace event recorded; non-transient errors skip fallback.

## Risks watch

- **Provider-specific exception types** — `langchain-openai` and `langchain-anthropic` may wrap HTTP errors differently. `classify_error` must handle both; test with realistic exception shapes.
- **Env var expansion security** — only expand `${VAR}` syntax, never shell-execute. The regex must be strict.
- **Cache memory** — adapters are cached indefinitely. Acceptable for V0.4; if agent count grows large, add LRU in V0.5.
