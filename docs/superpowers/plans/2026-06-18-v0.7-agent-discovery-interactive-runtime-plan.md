# V0.7.0 Agent Discovery & Interactive Runtime — Implementation Plan

> Implement milestone by milestone with tests first. Do not mix unrelated cleanup
> into this release. The design spec is the source of truth; if implementation
> pressure contradicts it, update and review the spec before changing behavior.

**Goal:** Make Agents discoverable and directly runnable by name, then provide an
opt-in native task/HITL protocol and generic CLI progress experience that replaces
Research Assistant's example-specific orchestration.

**Spec:**
[`2026-06-18-v0.7-agent-discovery-interactive-runtime-design.md`](../specs/2026-06-18-v0.7-agent-discovery-interactive-runtime-design.md)

**Tech stack:** Python 3.11+, `tomllib`, LangGraph, LangChain, Rich, pytest,
jsonschema, importlib metadata.

**Compatibility contract:** Existing direct API construction and
`modi run --agent NAME --agents-dir DIR --task FILE` remain supported. Task
protocol is opt-in and must not change existing Agents.
