# Harness API

Harness API is the public interface to Modi Harness.

See [`types-reference.md`](../types-reference.md) for `RunTaskRequest`, `RunTaskResponse`, `PendingApproval`.

## Operations

```text
run_task(agent, input, options) -> RunTaskResponse
resume_task(run_id, input) -> RunTaskResponse
approve_action(run_id, approval_id, decision) -> RunTaskResponse
reject_action(run_id, approval_id, reason) -> RunTaskResponse
get_state(run_id) -> AgentState
get_artifacts(run_id) -> list[WorkspaceRef]
get_trace(run_id) -> Iterable[TraceEvent]
get_denials(run_id) -> list[DeniedAction]

# memory
add_memory(record) -> MemoryRecord
list_memory(scopes, types, tags) -> list[MemoryRecord]
forget_memory(record_id) -> None

# threads (multi-turn)
start_thread(agent, options) -> ThreadInfo
end_thread(thread_id) -> None
list_threads() -> list[ThreadInfo]

# hooks (inspection)
list_hooks(run_id=None) -> list[HookSpec]
get_hook_result(run_id, hook_dispatch_id) -> list[HookResult]

# streaming variant
run_task_stream(agent, input, options) -> Iterator[StreamEvent]
```

See [`../types-reference.md`](../types-reference.md) for `ThreadInfo`, `StreamEvent`, `HookSpec`, `HookResult`.

## Threads

A `thread_id` carries memory scope `conversation` and chat history across multiple `run_task` calls. A run started without `thread_id` is a one-shot.

Threads are explicit: `start_thread` opens one, `end_thread` closes it (drops `conversation` memory). Without an explicit start, every `run_task` is independent.

## Streaming

`run_task_stream` returns an iterator of `StreamEvent` (model deltas, tool-call events, approval events, terminal). The terminal event carries the final `RunTaskResponse`. The non-streaming `run_task` is a thin wrapper that consumes the stream and returns the terminal payload.

## Rules

- API is thin. Runtime Adapter owns execution.
- Approval, denial, resume, state, artifact, trace, denial, and memory operations are explicit. Side effects are never hidden behind `run_task`.
- Denials are persisted for denied-retry checks.
- Python API is primary. HTTP and CLI wrap this API later.
- Responses are structured enough for CLI, notebook, and service callers.
- `run_id` is generated by Runtime Adapter; API never invents one.

## Boundaries

- Execution: Runtime Adapter.
- Memory persistence: Memory Store.
- Trace: Trace Recorder.
- Workspace: Workspace Manager.
