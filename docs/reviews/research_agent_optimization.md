# Modi Harness Research Agent 优化方法论

**主线：闭环 → 可观测 → 收敛 → 抽象 → 质量**

1. **闭环**：能完成任务、提交结果、正常结束。核心是 `submit_output`、校验与自动落盘。  
2. **可观测**：用 Trace、Token Breakdown、延迟与工具指标定位瓶颈，杜绝凭感觉优化。  
3. **先 Agent 后 Harness**：Agent 负责策略与执行，Harness 负责通用控制与工具，先发现能力问题再沉淀机制。  
4. **执行阶段化**：把自由推理变成固定流水线（PLAN → FETCH → EVIDENCE → SUBMIT），一阶段一任务，减少重复与犹豫。  
5. **工具阶段零表达**：执行阶段只做动作，不解释、不总结，降低模型输出与上下文污染。  
6. **Prompt 规则化**：用“必须/禁止/只允许/最多/不得”替代“应该/建议/目标是”，把 prompt 变成执行规则。  
7. **效率与质量分离**：当前已验证执行效率（行为收敛、工具调用收敛、可观测、输出稳定），研究质量（证据、来源、准确性、深度）待单独证明。

**最终总结**：  
先让 Agent 行为收敛，再让 Harness 机制抽象，最后独立验证研究质量。
